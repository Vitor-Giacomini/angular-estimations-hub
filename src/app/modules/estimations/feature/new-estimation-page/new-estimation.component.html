Estimation Instructions

- No fields can be left empty

If the savings are null, the value should be zero.

- The estimation's description should follow the Standard Declaration Pattern

E. g.: Change (part) material: XXXX to XXXX.

- Savings is always calculated on a per-unit basis

Same goes for the "Products" page.

- The estimation's status will always start as "proposed"

The status can only be changed on the "Estimations" page.

- Ensure that all the information is correct before submitting the estimation

Inaccurate or incomplete estimations will be promptly rejected.
